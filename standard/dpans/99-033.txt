Proposal 99-033:

Replace:
17.6.1.2191 SEARCH
STRING

        ( c-addr1 u1 c-addr2 u2 -- c-addr3 u3 flag )

Search the string specified by c-addr1 u1 for the string specified by
c-addr2 u2. If flag is true, a match was found at c-addr3 with u3
characters remaining. If flag is false there was no match and c-addr3 is

c-addr1 and u3 is u1.

With:

17.6.1.2191 SEARCH
STRING

 ( c-addr1 u1 c-addr2 u2 -- c-addr3 u3 flag )

Search the string specified by c-addr1 u1 ( the searched string) for the
first occurrence of the string specified by c-addr2 u2 ( the find
string). The search is to start from c-addr1. If flag is true, a string
matching the find string was found starting at c-addr3 with u3
characters remaining in the searched string (c-addr1+u1 always equals
c-addr3+u3). If flag is false no match was found and c-addr3 is c-addr1
and u3 is u1.

If u2 is zero (i.e. the find string is null), then flag is true, c-addr3
is c-addr1, and u3 is u1 (as if a match were found at the beginning of
the searched string). If u1 is zero (i.e. the searched string is null),
and  u2 is nonzero (i.e. the find string is not null), flag is false,
c-addr3 is c-addr1, and u3 is u1 (as if there were no match).

Michael Milendorf:

               THE PROPOSAL IN GENERAL IS FINE, BUT IT CREATES
               AN AMBIGUITY. IT WOULD BE BETTER IF I POINTED IT
               OUT DURING THE DISCUSSION PERIOD. THE SECOND
               PARAGRAPH STARTING FROM (If u2 is zero ... )
               IS AMBIGUOUS BECAUSE IT PRESENTS 2 ALTERNATIVE
               PATHS (CHECKING FOR U1=0 FIRST, OR CHECKING FOR U2=0
               FIRST) AN IMPLEMENTOR IS FREE TO CHOOSE. THEREFORE
               2 DIFFERENT IMPLEMENTATIONS OF "STRING" MAY
               RESULT IN DIFFERENT EFFECTS. THE DESCRIPTION
               MUST BE CLEAR IN WHO IS BEING CHECKED AGAINST
               ZERO *FIRST*: U1 OR U2. THERE IS NO USE OF TRYING
               TO FIX AN EXISTING AMBIGUITY BY PRESENTING A
               NEW ONE, THEREFORE I SUGGEST TO IMPROVE THE
               SECOND PARAGRAPH AND RESUBMIT THE PROPOSAL.

               [Greg observation:  As with Dave's comment, the
               above is simply untrue.  The second paragraph
               identifies the following cases:

                  a.  u2=0
                  b.  u2 nonzero and u1 zero
                  c.  All others (specifically, u1 nonzero).

               Case (a) exists regardless of the value of u1.
               There is no intersection between the sets a and b.
               There is no intersection between any pair of these
               three sets.  Hence it makes no difference what is
               tested first, and there is no ambiguity.  Several
               people have read this proposal in a hurry and have
               missed "and u2 is nonzero" in the second sentence
               of the second paragraph, but this definitely makes
               the cases of the first and second sentences mutually
               exclusive.]
