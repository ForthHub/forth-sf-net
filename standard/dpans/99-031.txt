Proposal 99-031:

Replace the definitive text for "16.6.1.2197 SET-ORDER" as follows:

Old:

Set the search order to the word lists identified by widn ... wid1.
Subsequently, word list wid1 will be searched first, and word list
widn searched last. If n is zero, empty the search order. If n is
minus one, set the search order to the implementation-defined minimum
search order. The minimum search order shall include the words
FORTH-WORDLIST and SET-ORDER. A system shall allow n to be at least
eight.

New:

Set the search order to the sequence of word lists identified by widn ... wid1.
Subsequently, word list wid1 will be searched first, and word list
widn searched last. If n is zero, empty the search order. If n is
minus one, set the search order to the implementation-defined minimum
search order. The minimum search order shall include the words
FORTH-WORDLIST and SET-ORDER. A system shall allow n to be at least
eight.

If the sequence widn ... wid1 contains duplicate entries, the search
order shall reflect such duplications to the extent that a subsequent
use of GET-ORDER shall return the same sequence widn ... wid1 that was
passed to SET-ORDER .  (This stipulation makes it possible to use
SET-ORDER and GET-ORDER to implement the semantics of 16.6.2.0715 ALSO
and 16.6.2.2037 PREVIOUS.)  A system is permitted, but not required,
to optimize the implementation of FIND so that such duplications do
not increase FIND's execution time since, during a single execution of
FIND , if a given definition is not located during the first search of
a particular word list, it will not be located during a redundant
search of the same word list.
