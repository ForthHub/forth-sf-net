Proposal 99-029:

In the definitive text for "11.6.1.2090 READ-LINE":

a) Replace the second and third paragraphs as follows:

Old:

If the operation succeeded, flag is true and ior is zero. If a line
terminator was received before u1 characters were read, then u2 is
the number of characters, not including the line terminator, actually
read (0 <= u2 <= u1 ). When u1 = u2 the line terminator has yet to
be reached.

If the operation is initiated when the value returned by FILE-POSITION
is equal to the value returned by FILE-SIZE for the file identified by
fileid, flag is false, ior is zero, and u2 is zero. If ior is
non-zero, an exception occurred during the operation and ior is the
implementation-defined I/O result code.

New:

"ior" is an error indicator.  If ior is zero, no error occurred during
the operation of READ-LINE, and the values of flag and u2 are as
defined below .  If ior is nonzero, an error occurred, ior is the
implementation-defined I/O result code, and both flag and u2 are
undefined.  An end-of-file condition is not considered to be an error
in this context.

"flag" is an end-of-file indicator.  As indicated above, its value
is defined only if ior is zero.  If flag is false, the file was positioned
at the end of the file (i.e. the values that would have been returned
by FILE-POSITION and FILE-SIZE are equal) just prior to the execution
of READ-LINE, and u2 is zero.  Otherwise flag is true and the value of
u2 is as defined below.

"u2" is the number of non-delimiter characters that were transferred
into the buffer.  If u2 is equal to u1, then u1 non-delimiter
characters were transferred into the buffer, but the process of
handling this line is not yet complete, and additional calls to
READ-LINE will be necessary to either read any additional characters
of that line or to determine that there were none.  If u2 is nonzero
and less than u1, then u2 non-delimiter characters, constituting the
entire line (or the entire remainder thereof, if the previous call
to READ-LINE returned u1), were transferred into the buffer.  If u2
is zero and flag is zero, the line (or the remainder thereof, if the
previous call to READ-LINE returned u1) was empty.

For a system where lines are ordinarily terminated by a sequence of one
or more delimiter characters, the lack of such a delimiter sequence at
the end of the last line of the file shall not be treated as an error
condition.  When applied to such a line, READ-LINE shall read the
characters of that line, as above, and the next call to READ-LINE after
all the characters have been read shall return the end-of-file
indication.

The following table summarizes the valid combinations of the return
values and their meanings (X means undefined):

 u2       flag   ior       Meaning
 ----     ----   ---       -------
 0<u2<u1  true   zero      The entire line (or remainder) was read

 0        true   zero      A blank line (or remainder) was read

 u1       true   zero      The first u1 characters of the line in the
                           file were read into the buffer, and subsequent
                           calls to READ-LINE will return the remainder
                           of the line.

 0        false  zero      No characters were read because the
                           end of the file was encountered

 X        X      nonzero   An error occurred.


b) Replace the last paragraph as follows:

Old:

At the conclusion of the operation, FILE-POSITION returns the next file
position after the last character read.

New:

At the conclusion of the operation, FILE-POSITION indicates the character
within the file to which the next call to READ-LINE will apply.  If u2
is less than u1, the file position will be after any implementation-defined
line-delimiter characters for the line that was read.  If u2 is equal to u1,
the file position will be just after the last non-delimiter that was
transferred into the buffer; in the case where the line contained exactly u1
characters, the file position just after the READ-LINE that returns u2=u1
will be just before any line-delimiter characters, and the file position will
not be advanced past those delimiters until the next READ-LINE .
